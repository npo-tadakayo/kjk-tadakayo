## 試算の前提・補足

### 前提条件

- **開発者スキル**: Webアプリ実務経験3〜5年。Firebase・Cloud Functions・Vanilla JSを使った開発経験あり。
- **含まれる工数**: 設計・コーディング・単体テスト・Firebase設定・デプロイ・軽微なデバッグ
- **含まれない工数**:
  - 要件定義・仕様確定のためのミーティング時間
  - コンテンツ（メールテンプレート文面・マスタデータ）の入稿作業
  - 運用後の追加改修・障害対応
  - Cloudflare / Firebase Console の初期設定（次田さん作業分）

### 試算の根拠

| 要素 | 判断理由 |
|---|---|
| Phase 1が最大の工数 | 認証・セキュリティルール・データモデル・基本CRMのベース構築が集中する |
| Phase 12が次点 | 認定事業所向けの別認証基盤・B2Bポータルは独立した開発規模 |
| Phase 5・6・8は比較的小規模 | 既存基盤（Cloud Functions・Firebase）を再利用できる |
| 幅（下限〜上限）が大きい理由 | UIのデザイン品質要求・テスト網羅度・ドキュメント作成量による変動 |

### Claude Code（AI支援開発）で内製する場合

本プロジェクトは Claude Code を用いた AI支援開発で進行中。AI支援により：

- コーディング速度が中級者比 **2〜4倍** に加速（推定）
- ただし設計・レビュー・テスト・Firebase設定は人間（次田さん）が確認必要
- 実質的な工数は外部委託の **1/3〜1/2** 程度に圧縮可能と見込む

| 区分 | 推定工数 |
|---|---|
| Claude Code 実装工数（実質） | 50〜80 h（セッション時間） |
| 次田さん確認・意思決定工数 | 20〜30 h |
| **合計（内製）** | **70〜110 h** |
